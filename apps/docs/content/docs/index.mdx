# Turborepo Example

A modern monorepo template built with **Next.js 16**, **React 19**, **Tailwind CSS v4**, and **TypeScript 5.9**. Powered by [Turborepo](https://turbo.build/) for fast, incremental builds across all apps and packages.

[GitHub Repository](https://github.com/owieth/turborepo-example)

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fowieth%2Fturborepo-example)

## Features

### Turborepo Caching

Turborepo remembers the output of every task it has ever run, so it never does the same work twice. This dramatically speeds up local development and CI pipelines.

### Shared UI Library (shadcn/ui)

A shared `packages/ui` library built on [shadcn/ui](https://ui.shadcn.com/) provides reusable components across all apps. Add a component once and use it everywhere.

### TypeScript Strict Mode

All apps and packages use TypeScript in strict mode with a shared base configuration in `packages/tsconfig`, ensuring type safety across the entire monorepo.

### Tailwind CSS v4

Styling is handled by Tailwind CSS v4 with a shared configuration, giving all apps a consistent design system out of the box.

### Nextra Documentation Site

A dedicated documentation app (`apps/docs`) built with [Nextra](https://nextra.site), providing a ready-to-use docs site with search, navigation, and MDX support.

### pnpm Workspaces

The monorepo uses [pnpm workspaces](https://pnpm.io/workspaces) for dependency management, ensuring fast installs and efficient disk usage through content-addressable storage.

## Quick Start

Clone the repository, install dependencies, and start the development server:

```bash
git clone https://github.com/owieth/turborepo-example.git
cd turborepo-example
pnpm install
pnpm dev
```

<Callout>
  Make sure you have [pnpm](https://pnpm.io/) installed. You can install it with `corepack enable` if you are using Node.js 16.13 or later.
</Callout>

This will start all apps in development mode. The web app runs on [localhost:3000](http://localhost:3000) and the docs site runs on [localhost:3001](http://localhost:3001).
